# Modern JavaScript for React Lab

Интерактивный учебный проект для темы `Модуль 0 / 3. Современный JavaScript для React`.

## Что покрывает тема

- `let` и `const` в роли реальных binding-стратегий в React-коде;
- стрелочные функции, шаблонные строки и выражения, которые формируют интерфейсную модель;
- `import/export` как реальный контракт между файлами проекта;
- destructuring, spread/rest, optional chaining и nullish coalescing;
- `map`, `filter`, `reduce`, работа с объектами и массивами;
- closures и то, как функции удерживают окружение;
- `Promise`, `async/await`, `fetch`, loading/error/success состояния;
- иммутабельность, различие мутации и копирования и влияние ссылок на предсказуемость React-состояния;
- то, как JavaScript-модели данных проявляются в интерфейсе, а не остаются только в теории.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню переключения лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы есть 6 практических лабораторий:

1. `Синтаксис и выражения`
   Показывает `let/const`, шаблонные строки, destructuring, spread/rest, optional chaining и `??` на одном наборе данных.
2. `import/export`
   Даёт живую модель ES-модулей через named exports, default export и реальный модульный контракт текущего проекта.
3. `Коллекции данных`
   Разбирает `map`, `filter`, `reduce`, объекты и массивы как источник карточек, метрик и breakdown-блоков интерфейса.
4. `Closures`
   Показывает, как функции запоминают окружение и почему уже созданные обработчики продолжают держать старое значение.
5. `Promise, async/await, fetch`
   Делает настоящий `fetch` в локальный JSON и показывает загрузку, ошибку, пустой ответ и повторную фильтрацию результата.
6. `Иммутабельность`
   Сравнивает скрытую мутацию и корректное immutable-обновление на двух соседних панелях.

Во всех лабораториях есть:

- интерактивные переключатели и сценарии;
- ссылки на файлы текущего проекта и листинги из него;
- блоки `до/после` там, где они действительно помогают увидеть разницу поведения;
- типичные ошибки;
- короткие объяснения того, где это встречается в реальном React-коде.

Важный принцип проекта: тема показывается не только в UI-демо, но и в самом коде приложения. Поэтому в ключевых и неочевидных местах есть комментарии, а концепции стараются быть реализованы в их естественной форме: через ES-модули, pure functions, цепочки работы с массивами, настоящий `fetch`, функции-closures и сравнение ссылок в React-состоянии.
