# Компоненты, props, children и композиция

Интерактивный учебный проект для темы `Модуль 1 / 8. Компоненты, props, children и композиция`.

## Что покрывает тема

- функциональные компоненты как независимые единицы интерфейса;
- `props` как входные данные компонента;
- `children` и вложенность компонентов;
- композицию и переиспользование компонентов;
- поток данных по дереву компонентов;
- влияние изменения props на поддеревья;
- проектирование API компонента;
- типичные анти-паттерны: мутация props, copy props to state и попытки «переделать» компонент изнутри.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Функциональные компоненты и props`
   Показывает повторное использование одного компонента с разными входными данными.
2. `Children и вложенность`
   Разбирает `children`, `aside`, `footer` и slot-based композицию.
3. `Поток данных по дереву`
   Визуализирует, как props проходят через несколько уровней компонентов.
4. `Композиция и переиспользование`
   Собирает разные экраны из одного набора повторно используемых строительных блоков.
5. `Проектирование API компонента`
   Сравнивает плохой boolean-soup API и более ясный вариантный API.
6. `Ошибки и анти-паттерны`
   Даёт живые sandboxes для мутации props и рассинхрона между props и локальным state.

На каждой лаборатории есть:

- живые переключатели и параметры;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- пояснения про реальные последствия архитектурных решений.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому здесь есть реальные функциональные компоненты, контейнеры с `children`, композиционные layout-компоненты, сравнение разных API и sandboxes с плохими паттернами работы с props.
