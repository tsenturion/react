# JSX, createElement и базовый механизм рендеринга

Интерактивный учебный проект для темы `Модуль 1 / 7. JSX, createElement и базовый механизм рендеринга`.

## Что покрывает тема

- что такое JSX и зачем он нужен в React;
- как JSX связан с `React.createElement(...)` и почему это важно для понимания рендера;
- выражения JavaScript внутри JSX;
- ограничения JSX и типичные ошибки;
- различия между HTML и JSX;
- `Fragment` и работа с несколькими siblings без лишней разметки;
- формирование React element tree до обновления DOM;
- влияние входных данных на итоговую структуру интерфейса.

## Что внутри

Проект остаётся одной учебной страницей:

- сверху вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок про чтение проекта и стек с версиями.

Внутри 6 лабораторий:

1. `JSX как описание UI`
   Показывает, как JSX строит интерфейс из данных и условий.
2. `JSX и createElement`
   Сравнивает одну и ту же карточку через JSX и через `React.createElement(...)`.
3. `Выражения и ограничения`
   Разбирает, какие конструкции JavaScript подходят для JSX, а какие нет.
4. `HTML vs JSX`
   Показывает различия `className`, `htmlFor`, camelCase props, `style` object и self-closing tags.
5. `Fragments и читаемость`
   Объясняет, когда лишний wrapper портит структуру и зачем нужен `Fragment`.
6. `Данные → описание → UI`
   Связывает фильтры данных, JSX mapping и React element tree до обновления DOM.

На каждой лаборатории есть:

- живые переключатели и параметры;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- типичные ошибки и короткие пояснения;
- комментарии в важных местах кода.

Важный принцип проекта: тема раскрывается не только в UI, но и в самом коде. Поэтому здесь есть отдельные JSX-компоненты, отдельная реализация на `React.createElement(...)`, собственный инспектор React element tree, реальные `Fragment`-сценарии и чистые модели данных для всех лабораторий.
