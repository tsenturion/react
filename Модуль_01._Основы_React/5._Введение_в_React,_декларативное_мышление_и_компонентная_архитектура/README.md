# React Intro, Declarative Thinking and Component Architecture Lab

Интерактивный учебный проект для темы `Модуль 1 / 5. Введение в React, декларативное мышление и компонентная архитектура`.

## Что покрывает тема

- что такое React и где он начинает быть полезным;
- декларативный и императивный подход к UI;
- React как библиотеку для построения интерфейса из компонентов;
- thinking in components и границы компонентной ответственности;
- UI как дерево компонентов;
- разбиение экрана на независимые части;
- одну и ту же задачу в обычном JavaScript и в React;
- переход от ручных изменений DOM к описанию интерфейса через данные, состояние и композицию.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню переключения лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы есть 6 практических лабораторий:

1. `Зачем React`
   Показывает, где React начинает окупаться: когда растёт количество UI-частей, shared state и повторяемых блоков.
2. `Декларативный UI`
   Сравнивает ручные DOM-шаги и модель `данные → вычисление → рендер`.
3. `Thinking in Components`
   Даёт выбрать стратегию деления экрана на компоненты и увидеть последствия монолита, balanced split и over-split.
4. `Component Tree`
   Разбирает дерево компонентов, владельца состояния и цепочку props на реальном feature-поддереве текущего проекта.
5. `Обычный JS vs React`
   Ставит рядом imperative DOM-preview и React-preview для одной и той же задачи фильтрации и выделения карточек.
6. `Данные, state и композиция`
   Показывает, как экран собирается из независимых частей и как layout становится декларативной композицией.

Во всех лабораториях есть:

- интерактивные переключатели и сценарии;
- ссылки на файлы текущего проекта и листинги из него;
- блоки `до/после` там, где они действительно помогают увидеть смысловую разницу;
- типичные ошибки;
- короткие пояснения того, как знание проявляется в реальном React-коде.

Важный принцип проекта: тема показывается не только в UI-демо, но и в самом коде приложения. Поэтому здесь есть реальные React-компоненты, отдельные presentational layers, pure models, composition layout и даже imperative DOM-блок на обычном JS, сделанный специально для честного сравнения с React-версией. В важных и неочевидных местах добавлены комментарии.
