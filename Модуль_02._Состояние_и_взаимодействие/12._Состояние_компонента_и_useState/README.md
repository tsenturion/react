# Состояние компонента и useState

Интерактивный учебный проект для темы `Модуль 2 / 12. Состояние компонента и useState`.

## Что покрывает тема

- что такое state и зачем он нужен компоненту;
- `useState`, чтение и обновление состояния;
- state as a snapshot;
- batching и queued updates;
- direct updates и functional updates;
- stale state и потерянные обновления;
- различие между текущим рендером и следующим;
- как пользовательские действия переводят интерфейс в новый UI через state.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Введение в state`
   Показывает базовую роль `useState` как локальной памяти компонента.
2. `State as a snapshot`
   Разбирает, почему `setState` не делает значение “сразу новым” в том же обработчике.
3. `Batching`
   Показывает несколько state-обновлений в одном действии и итоговый UI следующего рендера.
4. `Queued updates`
   Сравнивает `setCount(count + 1)` и `setCount(prev => prev + 1)` на реальной очереди обновлений.
5. `Stale state`
   Воспроизводит потерянные инкременты в delayed callback и functional update как решение.
6. `Action → UI`
   Связывает пользовательское действие, несколько state-срезов и итоговый интерфейс экрана.

На каждой лаборатории есть:

- живые переключатели и действия;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о последствиях выбранного паттерна работы со state.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому в проекте есть реальные компоненты с `useState`, настоящие очереди обновлений, delayed callback со stale closure и экраны, где несколько независимых state-срезов одновременно меняют UI.
