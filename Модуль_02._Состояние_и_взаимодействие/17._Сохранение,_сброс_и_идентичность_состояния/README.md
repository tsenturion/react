# Сохранение, сброс и идентичность состояния

Интерактивный учебный проект для темы `Модуль 2 / 17. Сохранение, сброс и идентичность состояния`.

## Что покрывает тема

- как React связывает состояние с позицией компонента в дереве;
- preserving and resetting state;
- влияние `key`;
- идентичность компонента;
- когда компонент сохраняет своё внутреннее состояние, а когда теряет его;
- как управлять этим поведением осознанно;
- почему перестройка дерева меняет lifecycle и поведение интерфейса;
- как `key`, component type и slot дерева влияют на local state;
- типичные ошибки с reorder списка, переносом subtree и случайным reset.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Позиция в дереве`
   Показывает, как state живёт у того же component type в том же slot.
2. `Условные ветки`
   Сравнивает reuse одного type и reset при смене component type.
3. `Key и reset`
   Показывает, как `key` создаёт новую identity boundary.
4. `Списки и identity`
   Разбирает stable id, index key и forced remount в динамическом списке.
5. `Перестройка дерева`
   Сравнивает CSS reorder и реальный перенос subtree между слотами.
6. `Практикум решений`
   Помогает выбрать паттерн preserve/reset под конкретную архитектурную ситуацию.

На каждой лаборатории есть:

- живые переключатели и действия;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о последствиях выбранного паттерна identity.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому в проекте есть реальные local-state sandboxes, `key`, mount/unmount журнал, сценарии reorder списка, intentional remount и отдельные pure models для архитектурных правил React identity.
