# Локальное, глобальное, URL- и серверное состояние

Интерактивный учебный проект для темы `Модуль 4 / 27. Локальное, глобальное, URL- и серверное состояние`.

## Что покрывает тема

- local state vs global state vs server state vs URL state;
- границы ответственности разных видов состояния;
- когда данные должны жить рядом с компонентом, когда выше по дереву, когда в роутере/URL, а когда в слое серверных данных;
- архитектурные trade-offs при выборе места хранения состояния;
- влияние выбора слоя состояния на сложность, повторное использование и отладку;
- причинно-следственную связь между типом данных и тем, как React-приложение должно ими управлять.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Local state`
   Показывает, почему временный draft и раскрытие лучше держать рядом с карточкой.
2. `Global state`
   Разбирает общий preference store и контрастирует его с локальными деталями UI.
3. `URL state`
   Связывает фильтры экрана с address bar, history и shared link.
4. `Server state`
   Показывает loading, error, cache, refetch и отличие серверного слоя от local UI state.
5. `Trade-offs`
   Даёт интерактивный советник по выбору подходящего места хранения состояния.
6. `Общая архитектура`
   Собирает local, global, URL и server state в одном экране и показывает их совместную работу.

На каждой лаборатории есть:

- живые переключатели и интерактивные сценарии;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о том, почему выбранный слой состояния здесь уместен или ошибочен.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому в уроке есть настоящий global context-store, URL hook на `history.pushState`, отдельный слой server-state cache и интегрированный экран, где все виды состояния живут одновременно, но не смешивают свои роли.
