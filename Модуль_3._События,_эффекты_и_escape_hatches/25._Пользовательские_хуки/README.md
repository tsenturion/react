# Пользовательские хуки

Интерактивный учебный проект для темы `Модуль 3 / 25. Пользовательские хуки`.

## Что покрывает тема

- custom hooks;
- архитектура custom hooks;
- композиция;
- контракт hook-а;
- переиспользование логики между компонентами;
- инкапсуляция поведения;
- naming convention и границы абстракции;
- случаи, где custom hook не нужен;
- превращение шума из состояния, эффектов и обработчиков в читаемую модель.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Контракт`
   Показывает, как hook возвращает входы, команды и derived data на примере каталога.
2. `Композиция`
   Разбирает composed hook, который собирает фильтры, persistent preference и историю выбора.
3. `Переиспользование`
   Показывает, как один и тот же hook даёт изолированное состояние нескольким карточкам.
4. `API hook-а`
   Разбирает explicit command API вместо прямой утечки внутренних setter-ов.
5. `Границы`
   Показывает, когда hook оправдан, когда лучше helper, а когда логика должна остаться inline.
6. `Очистка шума`
   Сводит тему к практическому refactor формы через custom hook.

На каждой лаборатории есть:

- живые переключатели, формы и действия;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о последствиях выбранного подхода.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому здесь есть реальные `src/hooks`, composed hooks, persistent preference через `localStorage`, explicit command API, reusable disclosure-поведение и сценарий, где custom hook intentionally не создаётся, потому что достаточно pure helper.
