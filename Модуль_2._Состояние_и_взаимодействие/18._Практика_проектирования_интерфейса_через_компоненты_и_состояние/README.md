# Практика проектирования интерфейса через компоненты и состояние

Интерактивный учебный проект для темы `Модуль 2 / 18. Практика проектирования интерфейса через компоненты и состояние`.

## Что покрывает тема

- как разложить экран на компоненты, данные, состояние и связи между ними;
- как находить источник истины;
- как отличать данные, вычисления и события;
- как из текстового макета перейти к реактовской архитектуре;
- как действие пользователя меняет state, вызывает повторный рендер и приводит к визуальному результату;
- как проектировать один экран так, чтобы list, details и summary не расходились;
- почему derived values лучше вычислять, а не хранить отдельно;
- как проверять архитектуру на живом mini-demo, а не только в теории.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Декомпозиция экрана`
   Показывает, как один экран разбивается на component boundaries и уровни детализации.
2. `Источник истины`
   Сравнивает единый owner state и плохие варианты с duplicated selection и local draft.
3. `Данные и вычисления`
   Разбирает data, state, derived values и events на одном экране.
4. `Бриф в архитектуру`
   Превращает текстовый макет в список компонентов, state-срезов, derived values и событий.
5. `Цикл действия`
   Показывает цепочку `action → state → render → UI` на живом workbench.
6. `Итоговый mini-demo`
   Собирает все решения в цельный экран `CourseWorkbench`.

На каждой лаборатории есть:

- живые переключатели и действия;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о последствиях выбранной архитектуры экрана.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому в проекте есть реальный экран `CourseWorkbench`, вокруг которого построены sandboxes для source of truth, data classification, blueprint planning и live trace рендера.
