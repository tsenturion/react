# Поднятие и совместное использование состояния

Интерактивный учебный проект для темы `Модуль 2 / 15. Поднятие и совместное использование состояния`.

## Что покрывает тема

- lifting state up;
- shared state;
- единый источник истины;
- синхронизация нескольких компонентов;
- передача данных вверх и вниз;
- рассинхронизация при дублировании;
- prop drilling как следствие архитектуры дерева;
- как определить владельца состояния;
- как одно и то же состояние влияет на несколько визуально независимых частей интерфейса.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Lifting state up`
   Показывает общий parent state для двух синхронизированных контролов.
2. `Shared state`
   Разбирает один filter state для toolbar, grid и summary.
3. `Рассинхронизация`
   Сравнивает siblings с локальными копиями выбора и siblings с одним shared selectedId.
4. `Данные вверх и вниз`
   Показывает цепочку child → parent → sibling через callbacks и props.
5. `Prop drilling`
   Делает видимой глубокую цепочку прокидывания value и callbacks через дерево.
6. `Владелец состояния`
   Помогает определить, когда нужен local state, shared parent или более высокий owner.

На каждой лаборатории есть:

- живые переключатели и действия;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о последствиях выбранной архитектуры состояния.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому в проекте есть реальные parent-owned состояния, controlled child-компоненты, siblings с drift и без него, deep prop chain и decision model для выбора владельца состояния.
