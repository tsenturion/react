# Архитектура состояния: минимизация, производное состояние и нормализация

Интерактивный учебный проект для темы `Модуль 2 / 14. Архитектура состояния: минимизация, производное состояние и нормализация`.

## Что покрывает тема

- минимальное состояние;
- derived state;
- normalization;
- colocated state;
- где хранить состояние;
- когда состояние лишнее;
- когда данные лучше вычислять, а не хранить;
- как проектировать состояние так, чтобы оно не дублировалось и не расходилось;
- архитектурные решения для устойчивого UI;
- связь между структурой state и сложностью компонента.

## Что внутри

Проект построен как одна учебная страница:

- сверху общий вводный блок по теме;
- под ним меню лабораторий;
- в центре активная лаборатория;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы 6 лабораторий:

1. `Минимальный state`
   Показывает, что действительно нужно хранить, а что лучше производить каждый рендер.
2. `Derived state`
   Сравнивает вычисляемые totals и duplicated stored totals.
3. `Дублирование state`
   Показывает, как duplicated selectedTitle расходится с реальными данными.
4. `Colocated state`
   Разбирает local state в leaf-компоненте и unnecessary hoisting в parent.
5. `Где хранить state`
   Строит архитектурное решение из признаков данных и их владельца.
6. `Нормализация`
   Показывает normalization как способ убрать duplicated entities из нескольких веток.

На каждой лаборатории есть:

- живые переключатели и действия;
- ссылки на файлы текущего проекта;
- листинги из этих файлов;
- короткие пояснения о последствиях выбранной архитектуры state.

Важный принцип проекта: тема раскрывается не только в тексте, но и в коде самого приложения. Поэтому в проекте есть реальные React-компоненты с разными state-архитектурами, duplicated derived values, colocated leaf state, advisor по выбору владельца состояния и normalized entity storage.
