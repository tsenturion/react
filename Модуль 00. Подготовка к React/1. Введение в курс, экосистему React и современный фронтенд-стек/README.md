# React Ecosystem Lab

Интерактивный учебный проект для темы `Модуль 0 / 1. Введение в курс, экосистему React и современный фронтенд-стек`.

## Что покрывает тема

- что такое React в контексте современного фронтенда;
- как связаны браузер, DOM, JavaScript, Node.js, npm и пакетная экосистема;
- почему современный React почти всегда изучается вместе со сборкой, модулями и dev-инструментами;
- как устроен pipeline разработки от исходников до результата в браузере;
- чем отличаются подходы `без сборщика`, `через build tool` и `framework-first`;
- почему CRA больше не рассматривается здесь как базовая отправная точка для новых приложений;
- где в общей картине находятся Vite, React Router framework mode, Next.js и full-stack React.

## Что внутри

Проект построен как одна учебная оболочка с маршрутами лабораторий:

- сверху общий вводный блок по теме;
- под ним меню переключения лабораторий;
- в центре лаборатория, выбранная текущим URL-маршрутом;
- снизу общий блок с тем, как читать проект, и стеком с версиями.

Внутри страницы есть 5 практических лабораторий:

1. `Карта экосистемы`
   Показывает, где в реальности находятся браузер, DOM, JavaScript, Node.js, npm, Vite, React Router, React Router framework mode, Next.js, full-stack React и Docker. Внутри есть отдельный интерактивный блок про CRA, Vite, React Router framework mode и Next.js.
2. `Зачем React`
   Сравнивает ручные DOM-мутации и компонентную модель при росте сложности интерфейса.
3. `Pipeline`
   Даёт живую симуляцию пути `исходники -> transform -> dependency graph -> browser/runtime` и показывает, почему современная отправная точка сегодня объясняется через Vite/framework-first, а не через CRA.
4. `Подходы доставки`
   Сравнивает `no-build`, `Vite SPA` и `framework-first` по реальным требованиям проекта и связывает framework-first с React Router framework mode и Next.js.
5. `Node/npm/scripts/Docker`
   Позволяет провоцировать типовые ошибки окружения, сборки, тестов и контейнеризации.

Во всех лабораториях есть:

- интерактивные переключатели и сценарии;
- ссылки на файлы текущего проекта и листинги из него;
- состояния `до/после` там, где это действительно помогает увидеть смену модели;
- типичные ошибки;
- короткие объяснения, где это применяется на практике.

Важный принцип проекта: тема показывается не только в UI-демо, но и в самом коде приложения. Поэтому в ключевых и неочевидных местах есть комментарии, а концепции стараются быть реализованы в их естественной форме: через React-компоненты, состояние, чистые функции JavaScript и реальные инфраструктурные файлы проекта.

Отдельный архитектурный принцип этой темы: сам проект намеренно остаётся `Vite SPA` с обычным клиентским `React Router`. Так здесь можно прямо видеть client-side routing в коде текущего проекта, а React Router framework mode и Next.js разбирать как следующий слой на карте экосистемы, а не смешивать эти уровни.

## Запуск

```bash
npm install
npm run dev
```

Локально проект перепроверен на `Node.js 24.11.1`.

## Проверка

```bash
npm test
npm run build
```

## Preview

```bash
npm run preview
```

## Docker

```bash
docker compose up --build
```

После запуска приложение будет доступно на `http://localhost:8080`.
